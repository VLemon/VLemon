EventLoop是啥

通俗一点说就是一个特定条件下不会的结束的循环。一般来讲一个线程只能执行一个任务，任务执行完之后就退出，在特定的条件下我们需要一个不退出的线程来重复执行相同的逻辑或者检测等待一个异步操作的返回值就需要在线程当中写一个while(true)的循环来实现防止线程退出，通常的逻辑代码是这样的：

关于线程的阻塞状态
CPU并不是一直都执行一个进程的代码，而是在多个进程以及线程中快速的来回切换，达到并行的目的，调度器队列中的线程就是CPU当前需要执行的线程，当调用get_next_message()方法时 该线程会进入阻塞状态，阻塞状态其实就是该线程从CPU调度队列中移除，直到某些事件发生时在添加到CPU调度队列中。可参考 阻塞访问原理——等待队列

关于iOS中的Runloop

在iOS中Runloop实际上就是一个对象，是对事件循环封装的一个对象，这个对象管理了其需要
处理的事件和消息，并提供一个特定的入口函数来执行EventLoop循环里面的逻辑，线程执行到这个函数后会一直处于这个函数内部 接收 -> 等待 -> 处理 的循环中，直到这个循环结束。

OSX/iOS 系统中，提供了两个这样的对象：NSRunLoop 和 CFRunLoopRef。
CFRunLoopRef 是在 CoreFoundation 框架内的，它提供了纯 C 函数的 API，所有这些 API 都是线程安全的。
NSRunLoop 是基于 CFRunLoopRef 的封装，提供了面向对象的 API，但是这些 API 不是线程安全的。

CFRunLoopRef 的代码是开源的，你可以在这里 http://opensource.apple.com/tarballs/CF/CF-855.17.tar.gz下载到整个 CoreFoundation 的源码。

RunLoop 对外的接口以及Models

在CoreFoundation中总共有

Runloop 中对象的关系图

一个 RunLoop 包含若干个 Mode，每个 Mode 又包含若干个 Source/Timer/Observer。每次调用 RunLoop 的主函数时，只能指定其中一个 Mode，这个Mode被称作 CurrentMode。如果需要切换 Mode，只能退出 Loop，再重新指定一个 Mode 进入。这样做主要是为了分隔开不同组的 Source/Timer/Observer，让其互不影响。

CFRunLoopSourceRef 是事件产生的地方。Source有两个版本：Source0 和 Source1

  • Source0 只包含了一个回调（函数指针），它并不能主动触发事件。使用时，你需要先调用 CFRunLoopSourceSignal(source)，将这个 Source 标记为待处理，然后手动调用 CFRunLoopWakeUp(runloop) 来唤醒 RunLoop，让其处理这个事件。
  • Source1 包含了一个 mach_port 和一个回调（函数指针），被用于通过内核和其他线程相互发送消息。这种 Source 能主动唤醒 RunLoop 的线程，其原理在下面会讲到。

CFRunLoopTimerRef 时间事件触发的地方，NSTimer有时候会出现不准确的情况，原因是当执行Timer方法的时候会判断当前的时间是否已经过期如果过期就不会执行。

CFRunLoopObserverRef Runloop的每个状态都会通知Observer

关于定时器在不同的Mode

当我们创建定时器会作为一个item添加到kCFRunLoopDefaultMode(主线程大部分情况下会在这个Mode运行)这个Mode，但是一个runloop只能同时运行一个Mode里面的Item，所以当我们滑动tableview的时候主线程runloop的mode会切换到UITrackingRunLoopMode所以定时器就不会执行了。
当我们执行这个方法后：
[runloop addTimer:timer forMode:NSRunLoopCommonModes];
会把timer 的item 添加到当前Runloop的common mode items中，在该数组中的item会在所有的Runloop Mode的情况下都会执行。

关于定时器准时

NSTimer是不严谨的定时器，可能会出现不执行的情况（即使是在NSRunLoopCommonModes下）。原因得从Runloop的执行顺序入手。

可以看到Timer是进入循环后首先执行的，然后处理Source0 处理Source1，举个例子来说，如果timer的设置的时间间隔为0.1s 但是Source0 跟 Source1处理的时间超过了0.1s 那么下一次的Timer事件处理的时候会判断是否过期如果过期会跳过该次执行。（如果是在休眠状态到了Timer触发的时间会Runloop会被唤醒并执行Timer）。

Runloop实战

创建Runloop

创建Runloop的前提是要有一个线程，而且必须在线程内部创建，创建的方式很简单就是

NSRunLoop *runloop = [NSRunLoop currentRunLoop];

阻止线程退出

有时候我们需要一个子线程一直存在来执行相关方法，比如图片下载，这样就有一个需求在下载队列为空的时候需要将线程休眠，不在占用CPU资源。我们可以这么做

如果runloop没有执行体就会退出 所以我们给添加一个Source1事件
[runloop addPort:[NSMachPort port] forMode:NSDefaultRunLoopMode];
这样Runloop会在休眠状态不会占用CPU资源等待唤醒，当有Source0 或者 Timer事件的时候就会唤醒。
